what_broke: |
  Single-point HV definition is a simplification of Eq 6.42 — the
  thesis specifies E[Hypv(Ẑ_{t+1}^{(i)*})] which technically
  includes the EMFC's projected POPULATION (not just the single
  portfolio's expected position). The single-point version is a
  reasonable proxy for AMFC ranking because (a) all portfolios
  share the same baseline z_ref, and (b) the ordering of
  single-point HV correlates with full-population HV contribution
  for the front member at consideration.

  Honest scar: a strict §6.4 implementation would propagate the
  KF state forward and compute the full predicted Pareto-set HV
  per candidate. W18 should consider that upgrade if W17-5's
  -8.72% gap doesn't reverse with the simpler version.
what_youd_change: |
  The deterministic tie-break (np.argmax smallest index) is
  defensible but could be made more thesis-faithful by adding a
  secondary criterion (e.g., min-risk among ties → conservative
  DM bias). Deferred — single-criterion is honest for now.

  The all-zero/NaN fallback logs a warning but the warning likely
  spammed the W17-5 smoke if it fired per period. Filed
  inline-investigation for W18: count the warning instances in
  the W17-5 run to see if degenerate periods are common.

  Should have added per-period u_star_idx to the trace_csv (not
  just per-period dict). That would have given W17-5 a direct
  receipt for which portfolios were being selected as u*_{t-1}.
